# PRD-Skill 执行记录

## 用户原始需求

> "我想做一个电商App，需要登录注册、商品浏览、购物车、下单功能"

---

## 技能执行过程

### 阶段：需求澄清（必须先完成）

根据 prd-skill 的工作流程，用户需求比较模糊，需要先进行需求澄清，**不能直接开始写 PRD**。

### 技能询问的问题清单

#### 【必问问题】

| # | 问题 | 说明 |
|---|------|------|
| 1 | **电商App的具体形态是什么？** | 比如：综合电商（淘宝/京东模式）、垂直电商（只卖某一类商品）、社交电商（拼团/直播带货）、二手交易平台、社区团购？不同形态决定了核心功能差异 |
| 2 | **目标用户群体是哪些？** | 比如：Z世代年轻人、宝妈群体、职场白领、学生党等？不同用户对界面体验和功能需求差异很大 |
| 3 | **核心要解决的用户痛点是什么？** | 用户当前在什么场景下遇到问题？比如：想要便捷购物但现有App太复杂？想要买到便宜好货但信任度不够？ |
| 4 | **本次迭代必须包含的功能有哪些？（做什么）** | 您提到的登录注册、商品浏览、购物车、下单，哪些是这次**必须**要实现的？ |
| 5 | **暂不在这次迭代范围内的功能有哪些？（不做什么）** | 比如：支付功能、优惠券、评价系统、客服功能、物流跟踪等是否暂不考虑？ |
| 6 | **这些功能的优先级如何排列？** | 如果时间有限，P0（必须完成）的是什么？P1（重要）的是什么？ |
| 7 | **计划什么时候上线？** | 有没有明确的时间节点或里程碑？ |
| 8 | **如何衡量功能是否成功？** | 有没有具体的验收标准或KPI？比如：日活、转化率、订单量等 |

#### 【追问场景示例】

**关于登录注册**：
- 需要支持哪些登录方式？（手机号、邮箱、第三方登录如微信/Apple）
- 是否需要找回密码功能？
- 是否有游客模式？

**关于商品浏览**：
- 商品分类层级大概几级？
- 是否需要搜索、筛选、排序功能？
- 商品详情页需要展示哪些信息？（图片、价格、库存、规格、评价等）
- 是否需要收藏/分享功能？

**关于购物车**：
- 是否允许多个店铺的商品共存？
- 是否需要数量修改、规格选择、删除功能？
- 是否有促销提示（如满减、优惠券）？

**关于下单**：
- 支持哪些收货地址管理？
- 是否需要发票功能？
- 是否有订单状态追踪？（待付款、待发货、待收货、已完成）

## 技能建议

1. **先聚焦核心路径**：电商App最核心的是"浏览→加购→下单"这个主流程，可以先围绕这个路径做减法
2. **明确MVP范围**：第一版可以先做最核心的功能，如：商品列表页→商品详情页→加购→下单→订单列表，其他功能后续迭代
